Backend API requirements for this screen

Screen Info
Route
/portal
Auth
Required — redirect to /portal/login if no session
Page type
Server-side rendered (SSR) — data must be fresh on each load
API Endpoints Required
Next Cleaning (hero widget)
GET /api/orders/next?customer_id={id} Returns: order_date, start_time, end_time, status, cleaner_name, property_address, cleaning_type
Recent Orders
GET /api/orders?customer_id={id}&limit=3&sort=desc Returns: array of order objects with date, type, status, cleaner, has_photos flag
Recurring Schedule
GET CONDITIONAL /api/recurring?customer_id={id} Only called if customer_type = "recurring". Returns: frequency, day, time, assigned_cleaner, next_4_sessions[]
Property Summary
GET /api/properties?customer_id={id} Returns: address, building, room, has_vacuum, has_photos, consent_status
Book a Cleaning (CTA)
No API call — generates Calendly URL client-side from session data: calendly.com/tokyocleaner/booking?name={name}&email={email}&a1={phone}
UI States to Handle
STATE No upcoming cleaning
Hero card shows empty state: "No upcoming cleanings" + prominent Book Now button
STATE No property registered
Onboarding banner is shown. Property summary card is hidden. CTA says "Register Property" instead of "Book a Cleaning"
STATE No orders yet (first-time)
Recent Orders shows empty state illustration. Hero card shows "No upcoming cleanings."
STATE Express customer (non-recurring)
Schedule widget is hidden. Right column shows a "Quick Actions" card instead with: Book a Cleaning, Report an Issue, Contact Support
STATE Loading
Each widget shows a skeleton loader independently (not a full-page spinner)
Bilingual Notes
All labels include EN + JP. The active language is toggled via the language switch in the top bar. In production: use next-intl or i18n key maps — never hardcoded dual text.

JP text is ~40–60% longer. All widget widths must accommodate the longer Japanese strings without overflow.
Data Sources (Supabase)
SELECT * FROM orders WHERE customer_id = $1 ORDER BY order_date DESC LIMIT 3

SELECT * FROM properties WHERE customer_id = $1 LIMIT 1

SELECT * FROM recurring_agreements WHERE customer_id = $1
