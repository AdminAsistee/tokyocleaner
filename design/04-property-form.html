🔧 Property Registry Specs

Backend requirements for Phase 1

Mapped API Endpoint
POST /api/properties
Payload includes all text fields + array of Storage IDs for photos.
Data Mappings (Supabase)
address: Full concatenated string.
has_vacuum: Boolean from radio selection.
photos: Array of paths in property-photos/ bucket.
Photo Storage Policy
Bucket: property-photos
Naming: {customer_id}/{timestamp}_{label}.jpg
Access: Authenticated user RLS (can only see their own folder).
UX Requirements
Success State: After successful POST, show a global toast/success banner on the redirection target (Dashboard).
